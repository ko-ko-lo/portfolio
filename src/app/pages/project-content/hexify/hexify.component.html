1. Purpose & Goals

The primary purpose of this plugin is to enhance the workflow, making it faster and more efficient to implement designs in code. This project also serves as a valuable resource for fellow developers and designers, providing them with a tool that enhances collaboration and speeds up their development processes. Additionally, it's a showcase piece for my GitHub account and portfolio, demonstrating my commitment to continuous learning and skill enhancement in Figma Plugin Development.

2. Architecture

The UI Layer of Hexify's architecture enables user interaction, while Code.ts orchestrates the main processes, handling data retrieval and coordination with helper modules.

  • UI Layer (ui.html):
    The interface where users select color collections and initiate the generation of output files.
  • Main Logic (code.ts):
    Central controller that handles requests from the UI, coordinates tasks with helper modules, and communicates with the Figma API.
  • Helper Module (api.ts):
    Gathers and filters color collections, preparing data for dropdown options.
  • Helper Module (utilities.ts):
    Converts colors into Hexadecimal and SCSS, CSS, or JSON formats for the final output.
  • Figma API:
    Provides Hexify with access to color collections and variables directly from Figma.

Dropdown Population (on UI Initialization)

When Hexify's UI is ready, the plugin automatically fetches color collections from Figma. The Main Logic coordinates with the Figma API and api.ts to gather and filter the necessary color data. Once processed, the dropdown is populated, allowing users to select from the available color collections with ease.

Output Generation (on Button Click)

When the user clicks Generate Output, Hexify processes the selected colors into the desired format. The Main Logic directs utilities.ts to convert the colors, returning them as formatted code. Finally, the output is sent back to the UI, ready for download in SCSS, CSS, or JSON format.

3. Visual Design & User Flow

The interface is organized into distinct sections for the introduction, color collection selection, output format selection, and action buttons, allowing users to quickly locate what they need. High contrast between text and background colors enhances readability, and a clear typographic hierarchy guides users smoothly through the interface.

User Flow from opening the plugin to downloading the output

  • Opening the Plugin:
    • User opens Figma and selects the plugin from the menu.
    • The plugin interface loads, displaying all available color collections.
  • Selecting Color Collections:
    • User sees a list of color collections.
    • The user selects one or multiple collections.
    • A "Select All" checkbox provides the option to select all collections at once.
    • If no collections are available, a message is displayed.
  • Choosing Output Format:
    • User selects the desired output format (e.g., SCSS or CSS) from a single-choice selection.
  • Click “Generate Output”:
    • User clicks the CTA button to export the selected color variables in a file.
    • If the user attempts to export without a selection (color collection and/or format), an error message prompts them to make a selection.
  • Download File:
    • The plugin provides a downloadable file to the user.
