3. Redesigning the Navigation: A Smoother, More Intuitive Experience

As Open Stillness grew into a multi-section platform, a modern, user-friendly navigation was needed. The initial dropdown-based system created friction, making it harder for users to navigate seamlessly - especially on mobile. To enhance clarity, accessibility, and responsiveness, I redesigned the navigation into a two-level system that is intuitive, structured, and works beautifully across all devices.

From Dropdown Menus to a Seamless Navigation Flow

The initial dropdown navigation required extra clicks and hovering, which was not ideal for mobile users. The new system introduces:

  • Primary Navigation: Direct access to Meditation, Breathwork, and Yin-Style Yoga.
  • Section-Specific Subnavigation: Only appears when inside a section, ensuring a cleaner interface.
  • Breadcrumbs for Deeper Pages: Enhances usability for individual routines & deeper content.

This eliminates dropdown complexity while keeping navigation clear and structured.

Summary: Redesigning the Navigation

To create a modern and seamless experience, the navigation was redesigned from a dropdown-based system to a structured two-level navigation. This update improves clarity, accessibility, and usability across desktop and mobile.

5. Enhancing Accessibility: Invisible Improvements That Make a Big Difference

Accessibility is something you experience. It ensures that everyone, regardless of ability, can navigate, understand, and interact with ease.

For me, accessibility is not an afterthought - it's a mindset. Whether designing a UI or developing features, I always consider keyboard navigation, contrast ratios, and semantic structure from the very beginning. I also focus on reducing unnecessary friction - keeping interactions smooth, avoiding excessive animations, and ensuring that motion is subtle and purposeful, so users can engage comfortably without distractions.

I took a deep dive into refining every detail to make Open Stillness a place where everyone feels welcome.

The Smallest Details Make the Biggest Difference

Accessibility isn't only about visible changes, it's about invisible refinements that make the experience effortless for all users. Here's how I made Open Stillness more inclusive:

  • Perfected the page title: because even small improvements in screen reader announcements matter.
  • Improved semantic HTML: because structure should be meaningful, not just functional.
  • Refined the accordion component: Replaced <div>s with proper <button> elements inside <h3>, and added ARIA landmarks (role="region") to improve screen reader usability.
  • Ensured proper keyboard navigation & focus order: Making sure filters, pages, and modals don't disrupt the user's flow.
  • Added hidden headings: Because not all users rely on visual cues to understand structure.
  • Checked color contrast & live updates (aria-live="polite"): Improving clarity and interaction feedback.
  • Validated markup & tested rigorously with Axe DevTools, Lighthouse, and keyboard navigation testing.
