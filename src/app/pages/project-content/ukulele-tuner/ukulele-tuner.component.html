1. Purpose & Goals

Last year, I bought myself a beautiful Ukulele. The first thing I needed was a tuner. I was looking for something reliable, easy to use, and ideally, something that captured the spirit of the instrument.

I started browsing digital tuners, only to discover that:

  • Most tuners were designed for guitar players, with Ukulele support added as an afterthought
  • The design of many apps felt generic, lacking any connection to the Ukulele's Hawaiian roots
  • Nearly all options were native apps, no simple browser-based tool existed

So I asked myself:
"Is it even possible to build a Ukulele tuner that works entirely in the browser?"

The Goals

  • Create a tuner that works directly in the browser
  • Offer both automatic pitch detection and a manual mode
  • Build a user interface that feels light, musical, and inspired by the Ukulele's origin
  • Use the project as a learning journey into Web Audio APIs, pitch detection, and real-time UI feedback

2. UX/UI Design

The design of Aloha Tune was guided by two core goals: clarity and character. I wanted users to immediately understand how to use the tuner, while also feeling like they'd stepped into a space created just for the Ukulele.

Logo Design

The project began under the name “Lele Tuner” with a playful flea icon referencing the Hawaiian meaning of "Ukulele" (jumping flea). But as the concept matured, so did the design direction. The final identity, “Aloha Tune,” embraces the spirit of Hawaii more fully with a sunset-inspired logo and a "Night on the Beach" color palette.

Interface System in Figma

All interfaces were designed and prototyped in Figma, allowing for fast iteration and theming. To keep the project cohesive and scalable, I designed a mini design system in Figma. This includes:

  • Typography
  • Color tokens (including support for light/dark mode)
  • Theme support for light/dark mode
  • Reusable components like toast messages, toggles, and the note visualizer

The layout is intentionally minimal. The tuner should never distract from the task: playing music.

Behind the Strings: UX Logic

One of the trickiest UX moments is managing microphone access. To guide the user smoothly, a link to enable the microphone is shown as soon as the application loads.

In addition to general user flows, I mapped out specific interaction sequences, including the core Tuning Detection Flow that powers the app.

Tuning Detection Flow – From the User's Perspective

When a user opens Aloha Tune, they're invited to enable microphone access. Once the mic is active, here's what happens behind the scenes:

  1. Microphone Access
    You play a note on your ukulele. The app immediately begins listening through your microphone.
  2. Audio Input
    It hears the sound and starts analyzing the pitch. Using pitch detection, it calculates the frequency, essentially how fast the string is vibrating.
  3. Frequency Detection
    The detected frequency is compared to a dataset of known note values (like C4, E4, A4). Even if you're slightly off, it can tell what you're trying to play.
  4. Note Matching
    The tuner identifies the closest matching note.
  5. Tolerance Check
    It gives you feedback in real time: If you're close - great! If not, it highlights whether the string is too low or too high, so you know exactly how to adjust.
